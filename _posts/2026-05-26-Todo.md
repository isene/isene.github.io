There was this one itch yesterday; I've been using [Hyperlist](https://github.com/isene/hyperlist) for my todo list since 2007. And with [scribe](https://github.com/isene/scribe) now smoothly handling Hyperlists natively, I was really happy. But not quite overjoyed. While calendar events do show up on both my laptop and my phone via various cloud calendars, my Todo Hyperlist was stuck on my laptop.

But first I need to rewind a bit. A few days ago I added an AI feature into [kastrup](https://github.com/isene/kastrup) (my messenger hub that handles emails, Discord, Slack, Messenger, Instagram, RSS and more). By pressing `z` while on a message, I can add some context to the message if I want or just press Enter — and [Claude](https://claude.com/claude-code) will figure out what to do with it. It will add a reminder as a calendar event or it will add a todo item into the proper place in my Todo Hyperlist.

So far so good. But I need the todo items, the tasks, also on my phone. And I need a proper two-way sync between my laptop and my phone for this. Any changes on my laptop should be mirrored on my phone and vice versa. And it should work whether the other device is on or off.

Easy peasy. Just fire up [Claude](https://claude.com/claude-code) and ask it to set up [Syncthing](https://syncthing.net) with my [DigitalOcean](https://www.digitalocean.com) server as an intermediate always-on store and create a todo app for Android. Not using any existing todo app out there, because then I can't add or change the app just like I want it. I need full control over the tools I use.

In just an evening [tasks](https://github.com/isene/tasks) was born. A [Kotlin](https://kotlinlang.org) app that serves my workflow perfectly. I'd never made any real app like this before, but now I have one. And since I'm not creating this for anyone else, I get to tailor-make it to my desires. There is much freedom and joy in that.
